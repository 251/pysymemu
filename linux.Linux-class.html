sys_mmap2(self, cpu, address, size, prot, flags, fd, offset)

source code 

Creates a new mapping in the virtual address space of the calling process.

Parameters:
  • cpu - current CPU.
  • address - the starting address for the new mapping. This address is used as hint unless the flag contains MAP_FIXED.
  • size - the length of the mapping.
  • prot - the desired memory protection of the mapping.
  • flags - determines whether updates to the mapping are visible to other processes mapping the same region, and whether updates are carried through to the underlying file.
  • fd - the contents of a file mapping are initialized using size bytes starting at offset offset in the file referred to by the file descriptor fd.
  • offset - the contents of a file mapping are initialized using size bytes starting at offset offset*0x1000 in the file referred to by the file descriptor fd.
Returns: int
  • -1 In case you use MAP_FIXED in the flags and the mapping can not be place at the desired address.
  • the address of the new mapping.

Creates a new mapping in the virtual address space of the calling process.

Parameters:
